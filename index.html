Architecture

Three data sources, smart filtering, and zero manual intervention.

🏢

Greenhouse API

Pulls real-time job listings from 15 top companies (Jane Street, Stripe, SpaceX, etc.) via their free public Job Board API. No authentication required.

🔎

Targeted Web Search

Uses Serper.dev API with site-specific queries (e.g., site:careers.google.com intern) to find individual job postings on 20+ company career pages.

🧠

Smart Filtering

Multi-pass NLP classification: regex-based type detection (Quant, Intern, Discovery), US-only geolocation, and title/description context matching to exclude full-time roles.

⚙️

GitHub Actions CI/CD

Cron-scheduled daily pipeline runs scraping, deduplication, newsletter generation, and data commits — fully automated with zero downtime.

📨

HTML Email Newsletter

Mobile-responsive email digest with deadline urgency indicators, direct apply links, and type-coded badges. Sent via Gmail SMTP with app passwords.

📱

React Dashboard

Single-page app with search, filtering, calendar view, and preference configuration. Built with React + Tailwind CSS, hosted on GitHub Pages.

Key Features

Built to solve a real problem — never miss an internship deadline again.

Multi-Source Aggregation

Combines Greenhouse Job Board API, Lever API, and Google Search to cover 40+ companies across tech and finance.

Intern-Only Filtering

Two-pass filter checks titles and descriptions to include only internships, co-ops, datathons, and discovery programs — no full-time noise.

US-Only Geolocation

Automatically identifies and filters by US states, cities, and common location formats. Rejects non-US postings.

Deadline Tracking

Regex-based deadline extraction from unstructured text. Urgency-coded alerts in newsletter (red/yellow/green).

Daily Email Digest

Clean, mobile-responsive HTML newsletter with direct apply links, type badges, and upcoming deadline section prioritized at the top.

Zero Maintenance

GitHub Actions cron job handles scraping, dedup, email sending, and data commits daily. Fully hands-off after deployment.
